AUTHENTICATION & SECURITY

JWT Token:
  - Issued by POST /auth/login
  - Algorithm: HS256
  - Expiration: 1440 minutes (24 hours configurable)
  - Payload: {"sub": user_id, "role": "student"|"teacher", "exp": timestamp}

Password Security:
  - Hash algo: bcrypt (12 rounds)
  - Pre-truncated to 72 bytes (bcrypt hard limit - Bug 3 fix)
  - Tokens in Authorization: Bearer <token> header
  
Role-Based Access:
  - require_student() dependency - 403 if role != "student"
  - require_teacher() dependency - 403 if role != "teacher"
  - Automatic for all protected routes
